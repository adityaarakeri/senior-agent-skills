# Code of Conduct

## Our commitment

We want participation in this project to be useful, respectful, and safe for everyone, regardless of experience, identity, background, or role.

## Expected behavior

- Discuss ideas and code without attacking the person behind them.
- Give specific, actionable feedback.
- Assume good intent while remaining clear about harmful impact.
- Respect differing constraints, tools, and levels of experience.
- Accept correction and repair mistakes.
- Keep private information private.

## Unacceptable behavior

Harassment, discrimination, threats, sexualized attention, deliberate intimidation, doxxing, sustained disruption, and publishing another person's private information are not acceptable.

## Scope

This code applies in repository discussions, issues, pull requests, reviews, and other project spaces. It also applies when someone is publicly representing the project.

## Reporting and enforcement

Report conduct concerns privately to the repository maintainers. Do not open a public issue that exposes sensitive personal information. If no private maintainer channel is available, use GitHub's private vulnerability reporting flow in the repository's Security tab and state that the report concerns conduct.

Maintainers may remove content, request a correction, issue a warning, restrict participation, or permanently ban someone when necessary. Reports will be handled as privately and consistently as circumstances allow.

## Attribution

This policy is informed by common open-source community standards and is intentionally written for this repository.
