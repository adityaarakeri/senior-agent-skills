# Security Policy

## Supported versions

Security fixes are applied to the current default branch. This repository does not publish versioned software releases yet.

## Reporting a vulnerability

Do not open a public issue for a vulnerability.

Use GitHub's private vulnerability reporting flow:

1. Open the repository's **Security** tab.
2. Select **Advisories**.
3. Select **Report a vulnerability**.
4. Include reproduction steps, affected files, impact, and any suggested mitigation.

Useful reports for this project include unsafe installer behavior, path or symlink handling that can overwrite unintended files, leaked credentials, and skill instructions that could cause destructive or unauthorized actions.

If private vulnerability reporting is unavailable, contact a maintainer privately through GitHub before sharing details publicly.

## What to expect

A maintainer will acknowledge the report, investigate it, and coordinate disclosure based on severity and available maintainer time. Please allow a reasonable remediation window before public disclosure.

Do not access data that is not yours, degrade services, or use social engineering while researching a report.
